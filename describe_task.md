Ниже сухое ТЗ для разработчика, без поэзии про “умные часы меняют жизнь”. Просто что делаем, зачем и что должно получиться.

## Задача

Реализовать интеграцию Hermes с Garmin Connect для автоматического получения утренних данных восстановления пользователя и формирования утреннего recovery-брифа.

## Цель

Каждое утро Hermes должен автоматически получать данные из Garmin Connect, анализировать показатели сна и восстановления, формировать короткий утренний отчёт и отправлять его пользователю.

## Что делаем

### 1. Подключение к Garmin Connect

Использовать Python-библиотеку:

```bash
garminconnect
```

Репозиторий:

```text
https://github.com/cyberjunky/python-garminconnect
```

Установить зависимости:

```bash
pip install --upgrade garminconnect curl_cffi
```

Интеграция неофициальная, через Garmin Connect account. Это MVP-решение, не официальный Garmin Health API.

---

### 2. Авторизация

Нужно реализовать авторизацию в Garmin Connect:

```text
email + password + MFA при первом входе
```

Требования:

```text
- логин через Garmin account
- поддержка MFA-кода при первом запуске
- сохранение токенов локально
- повторное использование токенов без ручного логина
- обработка истёкших токенов
```

Токены хранить в защищённой директории, например:

```text
~/.garminconnect/
```

или в проектной директории secrets/storage, если Hermes работает на сервере.

---

### 3. Получение данных

Каждое утро нужно получать данные Garmin за текущую дату.

Минимальный набор данных:

```text
- sleep data
- sleep score, если доступен
- sleep duration
- resting heart rate
- heart rate data
- stress data
- Body Battery
- HRV data, если доступна
- Training Readiness, если доступна
```

Ожидаемые методы библиотеки для проверки:

```python
client.get_sleep_data(date)
client.get_stats(date)
client.get_heart_rates(date)
client.get_stress_data(date)
client.get_body_battery(date)
client.get_hrv_data(date)
client.get_training_readiness(date)
```

Важно: часть методов может не работать или не отдавать данные для конкретного аккаунта/часов. Нужно сделать устойчивую обработку ошибок по каждому источнику отдельно.

---

### 4. Нормализация данных

Сырые данные Garmin нужно преобразовать в единый формат для Hermes.

Ожидаемый внутренний формат:

```json
{
  "date": "2026-05-21",
  "source": "garmin_connect",
  "sleep": {
    "duration_minutes": 430,
    "score": 78,
    "start_time": "2026-05-20T23:40:00",
    "end_time": "2026-05-21T06:50:00"
  },
  "body_battery": {
    "current": 61,
    "overnight_charge": 42,
    "min": 19,
    "max": 65
  },
  "stress": {
    "avg": 24,
    "max": 62,
    "overnight_avg": 18
  },
  "heart_rate": {
    "resting": 56,
    "avg": 68
  },
  "hrv": {
    "status": "balanced",
    "value": 48
  },
  "training": {
    "readiness": 64,
    "recovery_time_hours": 12
  },
  "raw_available": {
    "sleep": true,
    "body_battery": true,
    "stress": true,
    "heart_rate": true,
    "hrv": true,
    "training_readiness": false
  }
}
```

Если каких-то данных нет, поле не ломает весь процесс. Hermes должен уметь сформировать бриф из частичных данных.

---

### 5. Генерация утреннего брифа

На основе нормализованных данных Hermes должен сформировать короткий текст.

Ожидаемый формат результата:

```text
☀️ Morning Recovery Brief

Сон: 7ч 10м, качество нормальное.
Body Battery: 61, ночное восстановление +42.
Стресс ночью: низкий.
HRV: в балансе.

Вывод:
Восстановление среднее/хорошее. Сегодня можно планировать обычную нагрузку, но без тяжёлых интервальных тренировок утром.

Фокус дня:
Сложные задачи лучше ставить на первую половину дня.
```

Требования к брифу:

```text
- короткий
- понятный
- без медицинских диагнозов
- с практической рекомендацией по нагрузке
- с рекомендацией по рабочему режиму
- максимум 700–1000 символов
```

---

### 6. Расписание запуска

Настроить cron/job:

```text
Europe/Lisbon timezone
```

Рекомендуемое расписание:

```text
07:30 — первая попытка
07:45 — повтор, если данных сна нет
08:15 — финальная попытка
```

Логика:

```text
- если бриф за сегодня уже создан — не создавать повторно
- если данных сна ещё нет — завершить попытку и ждать следующего запуска
- если есть минимальный набор данных — создать бриф
- если Garmin недоступен — залогировать ошибку
```

Минимальный набор для создания брифа:

```text
sleep data + хотя бы один показатель восстановления:
- Body Battery
- stress
- resting HR
- HRV
```

---

### 7. Дедупликация

Нужно исключить повторное создание брифа за один день.

Ключ дедупликации:

```text
user_id + date + brief_type
```

Пример:

```text
user_id = main_user
date = 2026-05-21
brief_type = garmin_morning_recovery
```

Если запись уже есть, повторный запуск ничего не создаёт.

---

### 8. Доставка результата

На MVP достаточно отправлять бриф в Telegram.

Ожидаемый канал:

```text
Hermes → Telegram bot → user
```

Дополнительно можно сохранить результат в файл/БД:

```text
/data/hermes/briefs/garmin_morning_2026-05-21.json
```

или таблицу:

```text
morning_briefs
```

Структура записи:

```json
{
  "date": "2026-05-21",
  "type": "garmin_morning_recovery",
  "source": "garmin_connect",
  "metrics": {},
  "brief_text": "...",
  "created_at": "2026-05-21T07:46:00+01:00",
  "status": "created"
}
```

## Ожидаемый результат

После реализации:

```text
1. Hermes автоматически запускается утром.
2. Получает свежие данные из Garmin Connect.
3. Проверяет, достаточно ли данных для отчёта.
4. Нормализует recovery-метрики.
5. Генерирует короткий утренний recovery-бриф.
6. Отправляет его пользователю в Telegram.
7. Не создаёт дубли в течение одного дня.
8. Логирует ошибки авторизации, отсутствия данных и недоступности Garmin.
```

---

## Критерии готовности

Интеграция считается готовой, если:

```text
- авторизация Garmin Connect работает
- токены сохраняются и повторно используются
- можно получить данные сна за текущий день
- можно получить минимум 2–3 recovery-метрики
- данные нормализуются в единый JSON
- бриф создаётся автоматически утром
- бриф отправляется в Telegram
- есть retry-логика
- есть дедупликация
- ошибки не ломают весь процесс
```

---

## Важные ограничения

```text
- Используется неофициальная библиотека Garmin Connect.
- Garmin может изменить авторизацию или API.
- Интеграция подходит для личного MVP, не для коммерческого продукта.
- Не все метрики могут быть доступны для конкретной модели часов.
- Training Readiness / HRV / Body Battery нужно проверить на реальном аккаунте.
```

## Приоритет реализации

```text
P0:
- авторизация
- получение sleep data
- получение body battery / stress / heart rate
- сохранение raw JSON
- генерация простого брифа
- отправка в Telegram

P1:
- HRV
- Training Readiness
- retry-логика
- дедупликация
- нормализация данных

P2:
- красивое форматирование
- история брифов
- сравнение с предыдущими днями
- рекомендации по тренировкам и рабочей нагрузке
```

Коротко для разработчика: **делаем не Garmin-приложение и не официальный Health API, а утренний серверный импорт данных через `python-garminconnect`, нормализацию и Telegram-бриф.** Это самый короткий путь к работающему результату, без ритуального танца вокруг Connect IQ.
