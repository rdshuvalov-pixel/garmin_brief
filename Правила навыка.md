Вот основные правила оформления навыка для Hermes Agent:
    
    Структура навыка
    
    1. Корневая директория: Название навыка должно быть в формате название-навыка.
       
    2. Файлы:
       - README.md: Описание навыка, инструкции по установке и использованию.
       - skills/SKILL.md: Основной файл с метаданными навыка, включая описание, авторов и версию.
       - scripts/: Скрипты, которые управляют поведением навыка. Важно, чтобы все скрипты были работоспособными.
       - src/: Основная логика и модули для навыка, организованные по подпапкам (например, models, storage, jobs и т.д.).
    
    3. Файл конфигурации: 
       - config.yaml в корне навык. Указывает параметры, используемые в навыке.
    
    Описание навыка (SKILL.md)
    
    yaml
    
    name: название-навыка
    description: "Краткое описание навыка."
    version: 1.0.0
    author: "Имя автора"
    license: MIT/Apache-2.0
    required_env:                # Опционально, список необходимых переменных окружения
      - VAR1
      - VAR2
    tags: [tag1, tag2]          # Опциональные теги для поиска навыков
    homepage: "URL проекта"      # Ссылка на репозиторий или документацию
    
    
    
    Ключевые моменты:
    
    - Комментарии: Используй комментарии в коде для пояснения логики.
    - Документация: Обеспечь хорошую документацию, чтобы пользователи могли легко понять, как использовать навык.
    - Код: Следуй стандартам кода, включая стиль именования и структуру каталогов.
    - Тестирование: Включай тесты для проверки функциональности навыка.
    - Примеры использования: Добавляй примеры в документацию, чтобы пользователи знали, как правильно настраивать и использовать навык.
    
    Разрешения и безопасность:
    
    - Убедись, что все ключи и конфиденциальные данные не включены в репозиторий.
    - Используй переменные окружения для хранения секретов. Включи примеры в файл .env.example.
    
    Установка:
    
    - Инструктируй пользователей по установке навыка: 
      - Склонирование из репозитория.
      - Установка зависимостей (например, через pip).
      
    Следуя этим правилам, ты сможешь создать навык, который будет легко инсталлировать и использовать пользователям! Если нужны конкретные примеры или объяснения по любому из этого, дай знать!
